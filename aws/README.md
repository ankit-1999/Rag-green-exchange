## Quick Start

### 1. Deploy (15-20 minutes)

Follow the steps in **`AWS-DEPLOYMENT-STEPS.md`**:

- Open AWS CloudFormation console
- Upload `rag-cloud-formation.yaml`
- Fill in parameters (Environment: "poc", ExpirationDate: "2026-07-06")
- Check the IAM acknowledgment box
- Click Create
- Wait 10-15 minutes
- Copy the output values

### 2. Enable Bedrock Models (1 minute)

- Go to Bedrock console
- Click "Model access"
- Enable:
  - `amazon.titan-embed-text-v2:0`
   - `amazon.nova-micro-v1:0` (recommended)
   - `amazon.nova-lite-v1:0` (optional fallback)

### 3. Test (5 minutes)

- Open browser: `http://<EC2PublicIP>:8000/health`
- Open Swagger UI: `http://<EC2PublicIP>:8000/docs`
- Try GET /health endpoint

### 4. Upload a Test Document (5 minutes)

- Create `test.txt` with sample content
- Upload to S3 bucket
- Call POST /documents/upload via Swagger UI
- Verify status: "indexed"

### 5. Monitor & Cleanup

- Check CloudWatch alarms for budget
- On Day 7 (2026-07-06): Delete stack to stop charges

## Common Commands

### Deploy
```powershell
# Via AWS Console (easiest - see AWS-DEPLOYMENT-STEPS.md)
# Or via CLI:
aws cloudformation create-stack `
  --stack-name greengrid-rag-ai-poc `
  --template-body file://rag-cloud-formation.yaml `
  --parameters ParameterKey=Environment,ParameterValue=poc `
  --capabilities CAPABILITY_NAMED_IAM `
  --region us-east-1
```

### Monitor Progress
```powershell
aws cloudformation describe-stacks `
  --stack-name greengrid-rag-ai-poc `
  --region us-east-1 `
  --query 'Stacks[0].StackStatus'
```

### Get Outputs
```powershell
aws cloudformation describe-stacks `
  --stack-name greengrid-rag-ai-poc `
  --region us-east-1 `
  --query 'Stacks[0].Outputs'
```

### Delete Stack
```powershell
aws cloudformation delete-stack `
  --stack-name greengrid-rag-ai-poc `
  --region us-east-1
```
